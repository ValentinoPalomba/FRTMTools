# CLI Usage (`frtmtools`)

The CLI mirrors most of the macOS app’s capabilities. Install it via Homebrew or by
building the repository.

```bash
brew tap valentinopalomba/frtmtools
brew install frtmtools
```

## Commands

| Command | Description |
| --- | --- |
| `frtmtools analyze <path>` | Run the analyzer on an IPA/APK. Emits a JSON summary and an HTML dashboard identical to the macOS view. |
| `frtmtools diff <old> <new>` | Compare two packages and highlight size deltas. |
| `frtmtools tips <path>` | Print only the optimization tips generated by `TipGenerator`. |

### Output Files

- `report.json` – Structured result containing metadata, category breakdowns, stripping
  info, and tips.
- `dashboard.html` – Interactive report with per-binary stripping tables, category
  charts, dependency graph, and manifest insights.

### Automation Tips

- Use `--output <dir>` to change where artifacts are stored.
- Pair with CI to archive dashboards as build artifacts for every release candidate.
